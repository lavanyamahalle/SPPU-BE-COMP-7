## **Unit 3: Blockchain Platforms & Consensus**

### **Q1: Bitcoin, Hyperledger, Blockchain Types & Algorithm**

**1. Bitcoin**

* **Definition:** Decentralized digital currency based on blockchain technology.
* **Key Features:**

  * Peer-to-peer network, no central authority.
  * Transactions recorded in a public ledger.
  * Limited supply (21 million coins).
* **Mining:** Miners solve PoW puzzles to validate transactions.
* **Advantages:** Security, transparency, low transaction fees.
* **Diagram:**

```
User A --> Transaction --> Miners --> Block Added --> Blockchain Ledger
```

**2. Hyperledger**

* **Definition:** Open-source enterprise blockchain platform.
* **Key Features:**

  * Permissioned blockchain.
  * Modular architecture (Consensus, Ledger, Smart Contracts).
  * Example: Hyperledger Fabric.
* **Use Cases:** Supply chain, banking, healthcare.
* **Diagram:**

```
Peers --> Ledger --> Chaincode(Smart Contract) --> Transaction Validation
```

**3. Types of Blockchain**

| Type       | Description                 | Example           |
| ---------- | --------------------------- | ----------------- |
| Public     | Open to anyone              | Bitcoin, Ethereum |
| Private    | Permissioned access         | Hyperledger       |
| Consortium | Controlled by multiple orgs | R3 Corda          |

**4. Consensus Algorithm (Example: Proof of Work)**

* **Process:** Miners solve complex puzzles; first to solve adds block.
* **Purpose:** Ensure distributed consensus, prevent double-spending.
* **Diagram:**

```
Transactions --> Puzzle Solving by Miners --> Validated Block --> Blockchain
```

**5. Byzantine Generals Problem**

* **Scenario:** Multiple generals must agree on a battle plan but some may be traitors.
* **Significance in Blockchain:** Ensures honest consensus despite malicious nodes.
* **Example:** PoW ensures consensus even if some miners act maliciously.

## **Unit 5: Ethereum, Smart Contracts, Swarm, Whisper**

### **Q5: Ethereum, Smart Contracts, Swarm & Whisper**

**1. Ethereum**

* Decentralized platform for smart contracts and DApps.
* **Features:**

  * Smart contracts.
  * ERC standards (ERC20, ERC721).
  * EVM executes contracts.
* **Networks:** Mainnet, Testnet, Private.
* **Diagram:**

```
User --> Transaction --> Smart Contract --> EVM --> Blockchain Update
```

**2. Smart Contracts**

* **Definition:** Self-executing contracts with code + rules.
* **Types:** Public, Private, Consortium.
* **Example:** Escrow service for online payments.
* **Gas:** Fee to execute contracts.
* **Diagram:**

```
Contract Code + Conditions --> Trigger Event --> Automatic Execution
```

**3. Swarm**

* **Purpose:** Decentralized storage for Ethereum.
* Stores DApp data, avoids central servers.
* **Diagram:**

```
User Data --> Swarm Nodes --> Decentralized Storage --> Retrieval
```

**4. Whisper**

* **Purpose:** Decentralized messaging platform.
* Sends encrypted messages between nodes.
* **Diagram:**

```
Sender Node --> Whisper Protocol --> Receiver Node (Encrypted)
```
